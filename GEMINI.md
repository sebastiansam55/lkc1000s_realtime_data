# Temtop LKC-1000S+ 2nd Gen Modbus RTU Reverse Engineering Log

This log chronicles the process and discoveries made while reverse engineering the **Temtop LKC-1000S+ 2nd Gen** handheld air quality monitor to map its serial interface.

---

## The Reverse Engineering Journey

### Phase 1: Serial Interface & Fuzzing
*   **Context:** The device mounts as a standard USB Mass Storage device when plugged into a PC. However, it also presents a Virtual COM port (`/dev/ttyACM0` or `/dev/ttyACM1` on Linux).
*   **Method:** We wrote a serial fuzzer (`fuzzer.py`) to sweep common ASCII query keywords (`READ`, `GET`, etc.) and hex wake-up prefixes at multiple baud rates.
*   **Result:** A hit was discovered when sending the packet:
    `FE 04 00 03 00 01 D5 C5` (returning `FE 04 02 00 06 2D 26`).
*   **Analysis:** This packet is a standard Modbus RTU Read Input Registers (Function Code `04`) request pointing to Register `3` at Slave Address `0xFE` (254). The CRC checksum `D5 C5` is correct for Modbus RTU.

### Phase 2: Analyzing the SPI Flash Dump
*   **Context:** The hardware board contains an external SPI flash chip, which we analyzed using a 4MB binary flash dump (`dump1.bin`).
*   **Result:** The flash dump did not contain executable firmware. Instead, it was formatted as a FAT filesystem containing the device's internal `HISTORY` folder and historical air quality log files in CSV format (e.g. `DATE,PM2.5(ug/m3),PM10(ug/m3)...`).
*   **Takeaway:** This confirmed that the microcontroller's main program runs on its internal internal flash, while historical log records are compiled and appended to the external SPI flash. We focused our reverse-engineering on live polling of the internal register space.

### Phase 3: The Firmware Bounds Bug & Bypass
*   **Context:** We wrote a scanner (`modbus_scanner.py`) to sweep registers `0–64` using Function Codes `03` and `04`.
*   **Discovery:** 
    *   Querying individual registers `0`, `1`, or `2` using Function Code `04` caused the device to immediately crash, disconnect, and reboot.
    *   Querying register `17` or higher using Function Code `03` or `04` individually caused the exact same crash.
*   **Analysis:** This is a classic firmware index/pointer boundary check bug. The developers mapped internal sensor data arrays to specific registers, but the lookup logic lacked safety boundaries. Querying unmapped indices triggered a pointer crash or HardFault inside the MCU's USB/serial stack.
*   **Bypass:** We discovered that querying the entire register block `0–32` (quantity `33`) using a single contiguous Modbus bulk read transaction (`FC 04, Reg 0, Qty 33`) bypasses the individual address crash handler. The device responds stably with a 71-byte frame containing all sensor parameters.

### Phase 4: Column Correlation and Time Alignment
*   **Method:** We wrote a parser (`correlate.py`) to match the log lines from our raw Modbus logger (`temtop_log.csv`) against the official history log files generated by the device (`20260618.CSV`) based on nearest timestamps.
*   **Result:** 
    *   **DateTime Blocks:** The device exposes two duplicate clock structures (Registers `0-5` and Registers `20/28-32`).
    *   **PM Concentrations:** PM2.5 (Register 6) and PM10 (Register 7) are stored as raw values scaled by `1/10` (e.g. `11` $\rightarrow$ `1.1 µg/m³`).
    *   **HCHO & TVOC:** HCHO (Register 9) and TVOC (Register 10) are stored as raw integers scaled by `1/1000` (e.g. `130` $\rightarrow$ `0.130 mg/m³`).
    *   **Environmental Sensors:** Temperature (Register 16) and Humidity (Register 17) are signed integers scaled by `1/100` (e.g. `7780` $\rightarrow$ `77.80`).
    *   **AQI:** The Air Quality Index is not present in the registers; the MCU expects client software to calculate it.

## Math & Algorithms Used

### 1. Modbus RTU CRC-16 Checksum
Used to authenticate packets:
```python
def calculate_crc(data: bytes) -> bytes:
    crc = 0xFFFF
    for byte in data:
        crc ^= byte
        for _ in range(8):
            if crc & 1:
                crc = (crc >> 1) ^ 0xA001
            else:
                crc >>= 1
    return bytes([crc & 0xFF, (crc >> 8) & 0xFF])
```

### 2. US EPA AQI Calculation
The script dynamically computes AQI using concentration breakpoints:
$$I = \frac{I_{high} - I_{low}}{C_{high} - C_{low}} (C - C_{low}) + I_{low}$$
Breakpoints are implemented for PM2.5 and PM10.
